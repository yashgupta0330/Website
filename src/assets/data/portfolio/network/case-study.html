Overview

A Centralized System for Network Issue Tracing?

This system is used in Service Centers to trace customer-reported network issues—like when Wi-Fi isn’t working. It helps the team quickly understand where the issue is occurring across a layered network chain: CEN → OLT → ODF → PAT → DSL. By visually navigating through these layers, support agents can pinpoint the faulty segment and take appropriate action—reducing resolution time and improving response efficiency.

Challenges

Handling Technical Depth While Keeping the UI Simple

Legacy System Limitations: The old system lacked real-time visibility, had outdated navigation, and made fault tracing time-consuming. The challenge was to redesign without disrupting existing workflows

Complex Network Layers: Visualizing five technical network layers clearly (CEN to DSL) while keeping the interface user-friendly for non-technical agents.

Live Fault Status: Ensuring real-time updates and color-coded status indicators for each layer in the chain, while maintaining performance.

Complaint Mapping: Integrating complaint data with network topology to automatically highlight the faulty segment, saving time for support teams.
