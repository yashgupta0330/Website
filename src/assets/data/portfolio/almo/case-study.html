Overview

Create and manage Azure DevOps work items directly inside Outlook

Almo is an Outlook add-in designed to help professionals create and manage Azure DevOps work items effortlessly while working in their inbox. The goal was to enable users to convert emails into actionable tasks without switching platforms ensuring continuity of work, improved productivity, and reduced context switching.

Challenges

Solving Productivity Gaps in a Constrained Interface

Support with New Outlook: The existing add-in was built for the old Outlook experience and no longer aligned with Microsoft’s new interface patterns and technical standards.

Limited Interaction Space: New Outlook side panels provide very limited width, making information hierarchy and layout precision critical.

Form Overload: Work-item creation involved complex forms with multiple fields, making quick actions feel time-consuming.

Need of consistent UX: The classic interface lacked clear grouping, visual hierarchy, and modern UI behaviors.

Purpose and Direction

ALMO simplifies work item creation in Outlook.

Our aim was to design an intuitive experience that enables users to create and manage Azure DevOps work items without leaving their inbox. By embedding task creation directly into Outlook, we focused on reducing friction, improving form usability, and maintaining context — so users can take action instantly without breaking their workflow.
